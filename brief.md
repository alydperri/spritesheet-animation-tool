# Brief

My partner and I co-run a two-person indie studio. I did hand-drawn art and skeletal animations using Spine for our first game. It was the right process then -- but it was slow, and I don't have that kind of time anymore. We have kids now. I have a full-time job. The way I used to make games doesn't fit my life.

So we're redesigning how we work. One shift: moving from skeletal animation to spritesheets, and using AI generation to make that even faster. The bottleneck isn't generation -- it's everything after: checking the grid, slicing frames, removing chroma backgrounds, normalizing pivots and baselines, building a GIF for review, exporting the final spritesheet, and cleaning up without losing the source. Done ad hoc it works once. Done across a project it breaks.

I need a folder-based specialist that turns an anchor image and an animation request into production-ready game assets: deterministic local processing for all cleanup stages, two human review checkpoints, and nothing left behind except the approved GIF and spritesheet.

This is a real tool for a real studio figuring out how to keep making games inside a life that got a lot fuller.
